# Security routing

ECC secure defaults are passive guardrails. `gstack-cso` is the security review authority for authentication, authorization, OAuth, JWT, RBAC, payment, webhook, public API, secrets, personal data, uploads, cryptography, dependencies, CI/CD, admin access, and multi-tenant boundaries.

After a concrete finding, select at most one narrow ECC security skill. Select one allowlisted Cybersecurity skill only when additional specialist depth is genuinely needed. Never run overlapping security skills for the same finding.

Risk acceptance must be an explicit user decision naming the approver and affected finding. It is never generated automatically.
